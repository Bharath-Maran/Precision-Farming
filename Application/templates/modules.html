ARCHITECTURE DIAGRAM

This architecture diagram clearly depicts how the minute details of this project. The set of architecture on the top explains the developer point of view. The set of architecutre on the bottom explains the user point of view.

DATA CLEANING

1. Crop dataset (crop_data) is imported
2. Soil dataset (soil_data) is imported
3. Crop dataset crop names are listed
4. Soil dataset crop names are listed
5. The common crop names in both dataset are chosen
6. Updated crop dataset (crop_merged) is exported
7. Updated soil dataset (soil_merged) is exported

DATA INTEGRATION

1. Crop dataset (crop_merged) is imported
2. Soil dataset (soil_merged) is imported
3. Both dataset are plotted
4. The NPK values are randomly +/- by 3 values in loop
5. The final dataset (crop_recommendation) is exported

MODEL TRAINING

1. Final dataset (crop_recommendation) is imported
2. Data is separated into features and target
3. Data is split for training and testing
4. Training and testing is carried out for 6 ML Algorithms
5. Their metrics are calculated
6. Models are saved as pkl file
